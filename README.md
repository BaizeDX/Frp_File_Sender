# 📁 FileP2P - 点对点大文件传输工具

[![Python](https://img.shields.io/badge/Python-3.8+-blue.svg)](https://www.python.org/)
[![License](https://img.shields.io/badge/License-MIT-green.svg)](LICENSE)

**FileP2P** 是一个基于 Python 的点对点大文件传输工具，专为解决微信/QQ 等聊天软件的文件大小限制而设计。支持局域网高速直连和 frp 远程穿透访问。

## ✨ 功能特性

- 🚀 **大文件传输**：无文件大小限制，支持 10GB+ 大文件
- 🔗 **局域网直连**：同一子网自动直连，速度可达 100MB/s+
- 🌐 **远程穿透**：集成 frp 隧道，随时随地远程访问
- 📂 **即放即享**：文件放入 `files` 文件夹即可分享，无需上传
- 🔑 **访问码保护**：可选设置访问码，保护文件安全
- 🔄 **断点续传**：支持 HTTP Range 请求，中断后自动续传
- 📱 **响应式 Web 界面**：手机、平板、电脑均可使用
- 🔍 **智能网络检测**：自动判断直连还是穿透
- 📡 **局域网发现**：UDP 广播自动发现同网设备
- 💾 **指定下载目录**：接收方可自定义下载位置
- 🧠 **连接记忆**：自动保存最近连接，刷新不丢失
- 🌐 **自动打开浏览器**：启动后自动打开 Web 界面

## 🚀 快速开始

### 环境要求

- Python 3.8+
- Windows / macOS / Linux

### 安装

```bash
# 1. 克隆或下载项目
cd Frp_File_Sender

# 2. 安装依赖
pip install -r requirements.txt

# 3. 创建共享目录
mkdir files

# 4. 放入你要分享的文件到 files 目录
# （用文件管理器复制进去即可）
```

### 启动

```bash
# Windows - 双击运行
start.bat

# 或命令行启动
python -m server.main --dir ./files --port 8848
```

启动后浏览器会自动打开 `http://localhost:8848`。

## 📖 用户手册

### 发送方操作

#### 1. 准备文件
将需要分享的文件复制到 `files` 文件夹中：

```
Frp_File_Sender/
└── files/
    ├── 电影.mp4
    ├── 照片.jpg
    └── 文档.pdf
```

#### 2. 启动服务器
双击 `start.bat` 或运行：
```bash
python -m server.main --dir ./files --port 8848
```

#### 3. 设置访问码（可选）
在发送页面：
- 打开「🔑 访问码保护」开关
- 输入自定义访问码，或点击「🎲 随机」自动生成
- 点击「✅ 应用」
- 将访问码通过其他渠道（微信/短信）告诉接收方

#### 4. 生成分享链接
点击「🚀 生成分享链接」按钮，页面会显示：
- **局域网地址**：如 `http://192.168.1.100:8848`
- **访问码**（如已设置）
- **文件列表**

#### 5. 分享给接收方
将局域网地址和访问码（如有）发送给接收方。

---

### 接收方操作

#### 1. 打开接收页面
在浏览器中打开发送方提供的地址，如：
```
http://192.168.1.100:8848
```
点击「📥 接收文件」进入接收页面。

#### 2. 连接服务器
- 在「服务器地址」输入框中已自动填入地址
- 如有访问码，在「访问码」输入框中输入
- 点击「连接」

#### 3. 浏览和下载文件
- 连接成功后显示文件列表
- 勾选需要下载的文件
- 点击「下载选中」
- 浏览器会自动下载文件到默认下载目录

#### 4. 高级功能
- **智能检测**：输入对方 IP，自动判断最优连接方式
- **自动刷新**：每 10 秒检查新文件
- **搜索过滤**：搜索框快速查找文件
- **排序**：按名称/大小/时间排序
- **下载目录**：点击「📁 更改」指定下载位置
- **断点续传**：下载中断后重新连接自动继续

## ⌨️ 命令行参数

```bash
python -m server.main [选项]
```

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--dir`, `-d` | 共享文件夹路径 | `./files` |
| `--port`, `-p` | 服务器端口 | `8848` |
| `--host` | 绑定地址 | `0.0.0.0` |
| `--no-discover` | 禁用局域网发现 | 关闭 |
| `--tunnel` | 强制启用 frp 隧道 | 关闭 |
| `--auto-tunnel` | 自动判断是否需要隧道 | 关闭 |
| `--no-browser` | 不自动打开浏览器 | 关闭 |
| `--check IP` | 检测与目标 IP 的连接方式 | - |
| `--scan` | 扫描局域网内的 FileP2P 设备 | - |
| `--log-file` | 日志文件路径 | `logs/server.log` |

### 使用示例

```bash
# 基本启动
python -m server.main

# 自定义端口和目录
python -m server.main --dir /path/to/files --port 9000

# 检测网络连接方式
python -m server.main --check 192.168.1.100

# 扫描局域网设备
python -m server.main --scan

# 不自动打开浏览器
python -m server.main --no-browser
```

## ❓ 常见问题

### Q1: 启动后浏览器没有自动打开？
A: 手动访问 `http://localhost:8848`。如果使用 `--no-browser` 参数则不会自动打开。

### Q2: 接收方看不到文件？
A: 确认文件已放入 `files` 文件夹，然后点击「刷新」按钮。

### Q3: 接收方下载速度慢？
A: 局域网内通常在 10-100MB/s。如果很慢，检查是否在同一网络，或使用 `--check IP` 检测连接方式。

### Q4: 无法从其他设备访问？
A: 
1. 确认防火墙允许 8848 端口
2. 确认设备在同一网络（WiFi）
3. 检查 Windows 防火墙设置

### Q5: 端口被占用？
A: 使用 `--port 9000` 更换端口。

### Q6: 大文件传输中断？
A: 支持断点续传，刷新页面重新下载即可自动继续。

### Q7: 如何远程访问？
A: 需要配置 frp 服务器：
1. 编辑 `config/frpc.ini`
2. 使用 `--tunnel` 参数启动
3. 将 frp 提供的公网地址发给接收方

### Q8: 访问码不生效？
A: 访问码只对外部访问生效，本机（localhost/127.0.0.1）访问不需要输入。
